# 🌿 Résumé du Projet — Serre Connectée (Greenhouse)
> Méthode : **Design Thinking + Agile (Trello + GitHub)**

---

## 🧠 PARTIE 1 — Design Thinking (5 étapes)

### 1. Empathize (Empathie)
- Étudier les besoins des agriculteurs
- Problèmes identifiés : stress thermique, sol sec, surveillance manuelle

### 2. Define (Définir)
- Problème principal : manque de surveillance + pas d'automatisation
- Question clé : **Comment automatiser la surveillance des plantes ?**

### 3. Ideate (Idéation)
- Techniques utilisées :
  - **Brainstorming** : générer un maximum d'idées (robot, app, IoT…)
  - **SCAMPER** : améliorer l'idée choisie (Substitute, Combine, Adapt, Modify, Put to other use, Eliminate, Reverse)

### 4. Prototype (Solution choisie)
Une **serre intelligente connectée** avec :
- Capteurs de température et humidité
- Transmission des données via WiFi
- Dashboard de visualisation
- Activation automatique de la pompe

**6 modules :**
| Module | Description |
|---|---|
| Authentification | Connexion sécurisée |
| Gestion capteurs | Collecte des données IoT |
| Gestion pompe | Arrosage automatique |
| Dashboard | Affichage en temps réel |
| Notifications | Alertes automatiques |
| Configuration | Paramétrage du système |

### 5. Test
- ✅ Points positifs : interface facile, dashboard clair, arrosage automatique utile
- ❌ Problèmes : graphiques compliqués, configuration difficile
- 🔧 Solutions : interface simplifiée, options plus accessibles

## 📋 PARTIE 3 — Gestion Agile avec Trello

### Étape 1 — Créer un compte Trello
- Aller sur [trello.com](https://trello.com)
- Créer un compte
- Cliquer sur **Create → Board**
- Nom du board : `Greenhouse Project`

### Étape 2 — Créer les colonnes (listes)
Dans le board, créer 3 colonnes :
1. **To Do**
2. **Doing**
3. **Done**

> Cliquer sur *Add a list*, écrire le nom, puis *Add list*

### Étape 3 — Créer le Product Backlog
Dans la colonne **To Do**, ajouter les cartes suivantes :

| Carte | Points |
|---|---|
| Login | 2 pts |
| Dashboard | 5 pts |
| Capteurs IoT | 3 pts |

### Étape 4 — Ajouter les User Stories
Cliquer sur chaque carte → **Description** → écrire la user story :

| Carte | User Story |
|---|---|
| Login | En tant qu'utilisateur, je veux me connecter avec un email et un mot de passe afin d'accéder à mon compte en toute sécurité. |
| Dashboard | En tant qu'utilisateur, je veux voir les données (température, humidité) en temps réel afin de surveiller mes plantes facilement. |
| Capteurs IoT | En tant qu'utilisateur, je veux connecter et gérer les capteurs IoT afin de collecter les données de ma serre. |
| Pompe | En tant qu'utilisateur, je veux activer ou désactiver la pompe à distance afin d'arroser mes plantes automatiquement. |
| Notifications | En tant qu'utilisateur, je veux recevoir des alertes lorsque le sol est sec ou la température est élevée afin de réagir rapidement. |
| Configuration | En tant qu'utilisateur, je veux configurer les paramètres (température et humidité) afin d'adapter le système à mes besoins. |

### Étape 5 — Créer un Sprint
- Créer une nouvelle colonne : **Sprint 1 (2 semaines)**
- Durée : 2 semaines (ex : du 4 mai au 18 mai)

### Étape 6 — Ajouter les tâches au Sprint
Glisser (drag & drop) **seulement 3 cartes** vers la colonne Sprint 1 :
- ✅ Login
- ✅ Dashboard
- ✅ Capteurs IoT

> ⚠️ On ne met pas toutes les tâches — un sprint est limité dans le temps.  
> Les autres tâches (Pompe, Notifications, Configuration) seront faites dans les prochains sprints.

### Étape 7 — Assigner les tâches aux membres
Cliquer sur chaque carte → **Members** → assigner un membre :

| Tâche | Membre |
|---|---|
| Login | Membre 1 |
| Dashboard | Membre 1 |
| Capteurs | Membre 1 |

### Étape 8 — Ajouter une date de fin (Due Date)
Cliquer sur une carte → **Dates** → choisir une date de fin (+14 jours) → **Save**

### Étape 9 — Estimer l'effort
Modifier le titre de chaque carte pour ajouter les points :
- `Login (2 pts)` → facile
- `Capteurs IoT (3 pts)` → moyen
- `Dashboard (5 pts)` → difficile
